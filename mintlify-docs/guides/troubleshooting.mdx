## `/gwd doctor`

The built-in diagnostic tool validates `.gwd/` integrity:

```
/gwd doctor
```

It checks file structure, referential integrity, completion state consistency, git worktree health, and stale DB-backed runtime records.

## Common issues

<AccordionGroup>
  <Accordion title="Auto mode loops on the same unit">
    **Cause:** Stale cache after a crash, or the LLM didn't produce the expected artifact.

    **Fix:** Run `/gwd doctor` to repair state, then `/gwd auto`.
  </Accordion>

  <Accordion title="Auto mode stops with 'Loop detected'">
    **Cause:** A unit failed to produce its expected artifact twice in a row.

    **Fix:** Check the task plan for clarity. Refine it manually, then `/gwd auto`.
  </Accordion>

  <Accordion title="command not found: gwd">
    **Cause:** npm's global bin directory isn't in `$PATH`.

    **Fix:**
    ```bash
    npm prefix -g
    echo 'export PATH="$(npm prefix -g)/bin:$PATH"' >> ~/.zshrc
    source ~/.zshrc
    ```

    **Workaround:** `npx gwd-pi` or `$(npm prefix -g)/bin/gwd`
  </Accordion>

  <Accordion title="Provider errors during auto mode">
    | Error type | Auto-resume? | Delay |
    |-----------|-------------|-------|
    | Rate limit (429) | Yes | retry-after or 60s |
    | Server error (500, 502, 503) | Yes | 30s |
    | Auth/billing | No | Manual resume |

    For transient errors, configure fallback models:
    ```yaml
    models:
      execution:
        model: claude-sonnet-4-6
        fallbacks:
          - openrouter/minimax/minimax-m2.5
    ```
  </Accordion>

  <Accordion title="Budget ceiling reached">
    Increase `budget_ceiling` in preferences, or switch to `budget` token profile. Resume with `/gwd auto`.
  </Accordion>

  <Accordion title="Auto mode says another session is running">
    GWD now derives active-session ownership from DB-backed worker and dispatch state rather than `auto.lock`. In most cases `/gwd doctor fix` clears stale runtime rows and the next `/gwd auto` resumes normally:
    ```bash
    /gwd doctor fix
    ```
  </Accordion>

  <Accordion title="Git merge conflicts on .gwd/ files">
    GWD auto-resolves conflicts on `.gwd/` runtime files. For code conflicts, the LLM attempts resolution. If that fails, resolve manually.
  </Accordion>

  <Accordion title="EBUSY / EPERM / EACCES on Windows">
    **Cause:** Antivirus, indexers, or editors briefly locking files during atomic rename.

    **Fix:** Re-run the operation. Close tools holding files open if the error persists. Run `/gwd doctor` to verify repo health.
  </Accordion>

  <Accordion title="Worktree isolation stopped working after upgrade to v2.45+">
    **Cause:** The default `git.isolation` mode changed from `worktree` to `none` in v2.45.0.

    **Fix:** Set `git.isolation: worktree` explicitly in your preferences:
    ```yaml
    git:
      isolation: worktree
    ```

    In a brand-new repo, make an initial commit first. Until a committed `HEAD` exists, GWD temporarily runs worktree isolation as `none`.
  </Accordion>

  <Accordion title="Node.js version or git not found at startup">
    **Cause:** GWD v2.45+ checks for Node.js >= 22 and git availability at startup.

    **Fix:** Install Node.js 22+ (24 LTS recommended) and ensure `git` is in your PATH.
  </Accordion>
</AccordionGroup>

## `/gwd forensics`

Full-access debugger for post-mortem analysis:

```
/gwd forensics [optional problem description]
```

Provides anomaly detection, unit traces, metrics analysis, doctor integration, and LLM-guided investigation.

## MCP client issues

Use `/gwd mcp` to check MCP server status and connectivity at a glance.

<AccordionGroup>
  <Accordion title="No configured servers">
    Verify `.mcp.json` or `.gwd/mcp.json` exists and parses as valid JSON.
  </Accordion>

  <Accordion title="mcp_discover times out">
    Run the configured command outside GWD to confirm the server starts. Check backend URLs and dependencies.
  </Accordion>

  <Accordion title="Local server works manually but not in GWD">
    Use absolute paths. Set required environment variables in the MCP config's `env` block.
  </Accordion>
</AccordionGroup>

## Recovery procedures

### Reset auto mode state

```bash
rm .gwd/completed-units.json
```

Then run `/gwd doctor` to refresh projections and `/gwd auto` to restart from current DB-backed state.

### Reset routing history

```bash
rm .gwd/routing-history.json
```

### Full state rebuild

```
/gwd doctor
```

Refreshes `STATE.md` and related projections from authoritative database state.
